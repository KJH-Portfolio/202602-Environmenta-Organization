## 1. 초기 데이터 삽입 현황

현재 시스템은 아키텍처 설계에 따라 정적 데이터와 동적 데이터를 구분하여 관리합니다.

| 테이블 그룹 | 데이터 건수 | 주요 내용 |
|---|---|---|
| **MEMBERS** | 3명 | 관리자 1, 테스트 유저 2 |
| **QUESTS** | 50개 | 데일리 친환경 퀘스트 목록 (탄소 절감 수치 포함) |
| **QUIZZES** | 60개 | 난이도별(Easy/Normal/Hard) 환경 상식 퀴즈 뱅크 |
| **ITEMS** | 130개 | 상점 판매용 뱃지, 배경, 칭호 (등급별 배치) |
| **ECO_SHOPS** | 동적 생성 | 공공 데이터 API 연동을 통한 실시간 저장 방식 |
| **COMMUNITY** | - | 시연 시 사용자 인터렉션을 위한 빈 상태 유지 |

## 2. 테스트 계정 정보 (Test Accounts)

시연 및 기능 테스트를 위해 준비된 계정입니다. 모든 비밀번호는 `BCrypt` 암호화되어 보호됩니다.

| 분류 | 아이디 (ID) | 비밀번호 (PWD) | 역할 (Role) | 비고 |
|---|---|---|---|---|
| **관리자** | `admin` | `admin` | ADMIN | 시스템 관리 및 신고/블라인드 처리 권한 |
| **일반유저 01** | `user01` | `pass01` | MEMBER | 탄소 포인트 및 에코트리 성장 데이터 보유 |
| **일반유저 02** | `user02` | `pass02` | MEMBER | 신규 퀘스트 수행 및 상점 이용 테스트용 |

## 3. 데이터 동기화 아키텍처 (Dynamic Sync)

파이널 프로젝트는 데이터 무결성과 최신성 유지를 위해 다음과 같은 **동적 동기화(Dynamic Sync)** 방식을 채택하고 있습니다.

- **에코샵 (ECO_SHOPS)**: 초기 DB에 고정된 데이터를 넣지 않고, 사용자가 맵에서 특정 카테고리를 클릭할 때 **서울시 공공데이터 API**에서 `contsId`를 식별하여 자동으로 로컬 DB에 싱크(Synchronize)합니다. 이를 통해 실제 존재하는 최신 장소 정보를 보장합니다.
- **포인트 지갑**: `MEMBER` 테이블에 인서트가 발생할 때, 데이터베이스 **트리거(Trigger)**가 작동하여 자동으로 2,000포인트가 충전된 `POINT_WALLET`과 1레벨 `ECO_TREE` 인스턴스를 생성합니다.

## 4. 데이터 초기화 가이드
- 프로젝트 실행 시 `FinalProject/init_db.sql` 스크립트를 통해 스키마 생성 및 기초 데이터(퀘스트, 아이템, 퀴즈)가 주입됩니다.
- DB 재구축이 필요한 경우 Docker 컨테이너 재실행 시 `init_db.sql`이 자동으로 재실행됩니다.
